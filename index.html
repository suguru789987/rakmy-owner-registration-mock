オーナー登録 + 店舗・店長・従業員管理（案3 ハイブリッド）

オーナーは全操作の上位互換。店長に店舗運営を委譲できるが、オーナー自身もすべてを直接操作可能。

案3 ハイブリッドモデルのポイント:
・オーナーは店舗追加・従業員追加・店長招待をすべて自分で実行可能
・店長を招待すれば、担当店舗の従業員管理を委譲できる（店長は任意）
・オーナーは全管理者・全従業員の登録情報に常にアクセス可能
・店長がいなくても店舗運営できる。店長は「委譲先」であり「必須」ではない

権限階層

オーナー（全権管理者）
  • 全店舗・全機能にアクセス
  • 店舗の追加・編集・削除
  • 店長の任命・招待
  • 従業員の直接追加（全店舗）
  • オーナーの招待
  • 給与・経営管理・外部連携
→
店長（店舗管理者）
  • 担当店舗のみ管理
  • 従業員の招待・管理
  • 勤怠承認
  • 店舗追加は不可
  • 他店舗へのアクセス不可
  • ※ 店長の配置は任意
→
従業員
  • 打刻操作
  • 自分の勤怠確認
  • 管理機能へのアクセス不可
A

新規会社オーナー登録

会社名 + オーナー情報のみでシンプルに登録

  • 会社名・オーナー情報入力
  • メール認証 → PW設定
B

オーナー / 管理者 招待

別のオーナーや店長を招待

  • 管理者一覧 → 招待モーダル
  • メール → PW設定
C

店舗管理 + 店長招待

オーナーが店舗追加・店長任命

  • 店舗一覧（カード）
  • 店舗追加 + 店長同時招待
D

従業員管理

オーナー / 店長どちらからも追加可能

  • D-1: オーナー視点（全店舗横断）
  • D-2: 店長視点（担当店舗のみ）
  • D-3: 招待 → D-4: PW設定
E

パスワード再設定

パスワードを忘れた場合の再設定フロー

  • E-1: メールアドレス入力
  • E-2: 確認コード入力
  • E-3: 新パスワード設定
F

初回ログイン後

各権限ごとのダッシュボード初回表示

  • F-1: オーナー初回ログイン
  • F-2: 店長初回ログイン
