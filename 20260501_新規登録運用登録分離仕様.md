## 1. 設計コンセプト

タイムレコード利用開始に必要な4項目（業態・ブランド／店舗／雇用区分／従業員）の登録について、**「新規登録モード」と「運用登録モード」を画面・UIフィードバック・順序強制度で完全分離** します。

### 設計原則

| 観点 | 新規登録モード | 運用登録モード |
|---|---|---|
| **目的** | 利用開始のための4項目必須登録 | 既存運用での追加・編集 |
| **順序** | 必須（① → ② → ③ → ④） | 任意 |
| **強制度** | 強誘導（CTAモーダル） | 軽量示唆（CTAトースト） |
| **画面** | `setup-*` 専用ページ（ウィザード式） | `*-settings` 通常設定画面（一覧+追加） |
| **UI** | クリーンなステップカード | 一覧表 + 「+ 追加」 |

## 3. 画面仕様

### 3.1 B-0 登録ガイド (`#b0`)

**役割**: 4項目登録の進捗可視化・順序提示・ステップ画面への誘導ハブ

**最上部の構成（共通）**:
- `b0-completed-banner`: done=4 のときに「🎉 初期登録は完了しています」を表示
- 登録方法選択カード `method-picker`: 「設定画面で1件ずつ登録」/「CSVで一括インポート（推奨）」を選択。初期表示は **CSV一括（推奨）**。

#### 3.1.1 manualモード（設定画面で1件ずつ登録）

`ow-card` 内に **4ステップカード（ow-step）** のみを縦並びで表示:
- 1: 業態・ブランド登録 / 2: 店舗登録 / 3: 雇用区分登録 / 4: 従業員登録
- 各カード: ステップ番号 + タイトル + 説明 + 前提登録 + 「登録済み: N件」 + CTAボタン
- CTAボタン: 「登録する →」（未完了） / 「再編集 →」（完了済み）
- ロック表示: 前ステップ未完了時は `is-locked` + ボタン disabled
- ❗ 進捗リング・パーセント・残数・ステップマーカーは **非表示**（2026-05-11 簡素化）

#### 3.1.2 importモード（CSV一括インポート）（推奨）

`import-panel` 内に以下を縦並びで表示:

1. **方式選択（分岐枠）**: 独立ボックスでピル選択
   - 📋 テンプレートで取り込み（推奨タグ）
   - 🔄 外部勤怠サービスから取り込み
2. **📋 インポートガイド** リスト:
   - テンプレートモード時: **Step 1〜4**（業態・ブランド / 店舗 / 雇用区分 / 従業員）
   - 外部連携モード時: **Step 1〜2**（Step 1=業態・ブランド(テンプレ補完) / Step 2=店舗・雇用区分・従業員(外部一括)）
   - 各行: 番号アイコン + タイトル + 状態バッジ（進行中/✓完了/🔒 Step N完了後）+ CTA
3. **📌 後追加項目（推奨）**: 管理者をテンプレで取り込み（任意・モード共通）
4. **🛠 デモ用シミュレーション**: 静的モック⇄動的モックの連携擬似ボタン

> ⚠️ **B案採用（2026-05-11）**: 「方式選択」は分岐選択（性質: ピル即時切替）でガイドリストとは別枠。ガイドリストは **実取込作業のみ**（性質: 画面遷移＋数分かかる）を Step 1 から番号付け。

**CTAボタン共通仕様**:
- 「インポート画面を開く →」 → setup-import-* に遷移
- 「外部連携画面を開く ↗」 → 別タブで rakmy-timerecord-import-spec/#saas-setup
- 「管理者をテンプレで取込 ↗」 → 別タブで rakmy-timerecord-import-spec/#tpl-setup
- 順序違反は `is-locked` でガード

**完了後（done=4）**:
- 最上部の `b0-completed-banner` のみで通知（インポートパネル内・ウィザード内の重複バナーは廃止）
- サイドバーから「📋 登録ガイド」項目は非表示

### 3.2 setup-* 新規登録ステップ画面（4種）

各画面共通レイアウト:
- **サイドバー**: 「📋 登録ガイド」+ 「初期登録ステップ」(①②③④進捗マーク付き)
- **メインエリア**:
  - 「← 登録ガイドへ戻る」リンク
  - 中央に **緑枠ステップカード**: STEP X/4 バッジ + タイトル + 説明 + 前提登録/件数情報 + 「+ 登録する」大型ボタン

**動作**:
- 「+ 登録する」 → 該当登録モーダル展開
- モーダル登録完了 → CTAモーダル（橋渡し）→ 次の setup-* へ

### 3.3 *-settings 運用登録画面（4種）

各画面共通レイアウト:
- **サイドバー**: タイムレコ通常メニュー（打刻記録/勤務時間集計/給与/設定セクション）
- **メインエリア**:
  - タイトル + 「+ 追加」ボタン
  - 一覧表（既登録レコードを表示）

**動作**:
- 「+ 追加」 → 該当登録モーダル展開
- モーダル登録完了 → トーストCTA（右上）→ 任意で次の登録に連鎖

### 3.4 b-step-modal-* 登録モーダル（4種）

各エンティティ専用の純粋な登録フォーム。新規/運用 共通で利用。

| モーダル | フィールド |
|---|---|
| ① 業態・ブランド | 業態select / ブランド名 / ブランドコード |
| ② 店舗 | 店舗名 / 所属ブランド select / 住所 / 営業時間 |
| ③ 雇用区分 | 区分名 / 種別 / 時給単位 |
| ④ 従業員 | 名前 / 電話 / メール / 雇用区分select / 所属店舗select |

**仕様**:
- 純粋な登録フォームのみ（前提充足チェック警告は表示しない）
- 「キャンセル」「✕」 → モーダルクローズ
- 「登録」 → 進捗保存 + モーダルクローズ + モード別CTA表示

### 3.5 CTAモーダル（橋渡しモーダル）

**新規登録モード時のみ表示**。中央オーバーレイで強誘導。

```

新しい◯◯を登録しました。

[次の登録に進む →]   [登録ガイドへ戻る]

| 登録項目 | メッセージ | 主CTA | 副CTA |

| ① ブランド | 続けて、このブランドの店舗も登録しますか？ | 店舗の登録に進む → | 登録ガイドへ戻る |
| ② 店舗 | 続けて、この店舗の従業員も登録しますか？ | 従業員の登録に進む → | 登録ガイドへ戻る |
| ③ 雇用区分 | 続けて、この区分の従業員も登録しますか？ | 従業員の登録に進む → | 登録ガイドへ戻る |

### 3.6 CTAトースト

**運用登録モード時のみ表示**。画面右上スライドイン。

```

**動作**:
- 8秒で自動消去
- マウスホバー → タイマー停止
- ホバー解除 → 3秒タイマーで再開

```js
const isInitialMode = b0LoadDone() < 4;
const isOperationalMode = b0LoadDone() >= 4;
```

### 5.3 順序ガード

- `setup-N` が `done + 1` 以外なら警告バナー表示 + ボタン disabled
- `setup-N` で N <= done なら「再編集」モードで動作
- `b0CompleteStep(N)` 内で N !== done+1 の登録は alert で阻止

## 7. 実装にあたり確認事項（エンジニア向け）

### 7.1 ルーティング

- `/setup/brand`, `/setup/store`, `/setup/emptype`, `/setup/emp` （新規登録モード）
- `/settings/brand`, `/settings/stores`, `/settings/emptypes`, `/settings/employees` （運用モード）
- 初回ログイン時は `/setup/brand` から自動開始
- 全項目登録完了後は `/setup/*` URLは無効化（→ `/dashboard` リダイレクト）

### 7.2 モード判定ロジック

- バックエンド側で「初期登録完了フラグ」を保持するか、各テーブルの件数で判定するか
- 推奨: `companies.onboarding_completed_at` カラムを追加し、4項目すべてに1件以上の時刻を保存

### 7.3 順序ガード

- フロント側ガード: 順序違反URLにアクセス → 正しいステップへリダイレクト
- バックエンド側ガード: 必要な前提が未登録なら API側で 422 エラー返却

### 7.4 CTAモーダルの中断UX

- 「登録ガイドへ戻る」を選んだ場合、`b0` 画面で進捗バー＋次ステップ強調で再開できる
- 中断状態を sessionStorage（or DBに）に保存

### 7.5 トーストの実装

- アニメーション: スライドイン（ease 350ms）
- ホバー停止: `mouseenter` でタイマークリア、`mouseleave` で 3000ms タイマー再設定
- アクセシビリティ: `role="status"` `aria-live="polite"` 推奨

### 7.6 招待メール送信（従業員登録）

- 登録時に自動送信ではなく、**従業員一覧の「招待送信」ボタンから手動送信** が確定仕様
- 招待メールテンプレート: 既存 C-4 を「招待送信ボタン経由」に修正済み

## 9. インポート機能との連携（重要）

### 9.1 2つのインポート導線

新規登録ガイドで「CSV一括インポート」を選択した場合（方式選択は分岐枠、Step番号外）、および運用後の再更新時、別リポジトリの **タイムレコード インポート機能** へ別タブで遷移する。

#### 導線1: 新規登録ガイドから（ガイド順インポート推奨）
- 起点: b0「CSV一括インポート」カード → 方式選択（テンプレ/外部）→ ガイドリスト Step 1 の「インポート画面を開く →」
- 推奨フロー（テンプレ）: ガイド順（Step 1 業態・ブランド → Step 2 店舗 → Step 3 雇用区分 → Step 4 従業員）
- 推奨フロー（外部）: Step 1 業態・ブランド(テンプレ補完) → Step 2 店舗・雇用区分・従業員(外部一括)
- 戻り先: 元の b0 タブで進捗確認

#### 導線2: 各設定画面から（運用時再更新）
- 起点: 各設定画面（brand-settings 等）ヘッダーの「📥 CSVインポート ↗」
- 用途: 任意項目の更新・追加（例: 新店舗10件を一括追加）
- 戻り先: 元の設定画面タブ

### 9.2 連携先

- リポジトリ: `suguru789987/rakmy-timerecord-import-spec`
- 公開モック: https://suguru789987.github.io/rakmy-timerecord-import-spec/
- 機能: テンプレート取り込み / 外部勤怠サービスから取り込み（KoT・ジョブカン）

### 9.3 ⚠️ 動的・静的モック連携の課題

| | 新規登録ガイド | インポート機能 |
|---|---|---|
| モック性質 | **静的**（画面遷移のクリック動作のみ） | **動的**（CSV取込・データ反映） |
| 状態管理 | sessionStorage（モック内のフェイク） | 取込シミュレーション |
| リポジトリ | rakmy-owner-registration-mock | rakmy-timerecord-import-spec |

**課題**: 別リポジトリの sessionStorage は共有されないため、インポートモックでの取込完了が新規登録ガイドの進捗に自動反映されない。

### 9.4 本番実装で必要な連携要件

#### A. インポート完了通知
インポートジョブ完了時、バックエンドから登録ガイド側へイベント通知:
- 推奨方式: Webhook / WebSocket / Server-Sent Events / ポーリング
- 通知内容: 完了したエンティティタイプ、件数、エラー件数

```
{
  "event": "import.completed",
  "entity": "brand",
  "success_count": 3,
  "error_count": 0,
  "completed_at": "2026-05-11T11:30:00+09:00"
}
```

#### B. 登録ガイドの進捗自動更新
完了通知を受信したら:
- b0 の進捗バー / 件数 / ステップマーカーを自動更新
- 必要な4項目すべて充足したら完了サマリ画面に切替

#### C. 設定画面リストのリフレッシュ
各設定画面（brand-settings 等）で:
- 取込完了通知を受信 or タブフォーカス復帰時に一覧を再取得
- 取込されたレコードを即座に表示

#### D. エラー時の処理
- インポート画面側でエラー詳細を確認できる
- 登録ガイド側には「インポートが部分失敗しました」のステータス通知
- 再インポート導線を維持

#### E. 認証・セッション
- 同一テナントのオーナーがインポート画面とガイド画面の両方を操作する前提
- セッショントークンの共有または別ブラウザタブでのSSO

#### F. URLパラメータによる方式・ステップのスキップ（重要）

インポート画面へは **2つの起点で挙動を分ける**:

##### F-1. 新規登録ガイドからの起点（方式選択スキップ）

b0 の **方式選択（分岐枠）** で方式を選択済みなので、インポート画面では **方式選択画面をスキップして直接該当ステップへ遷移**。

URL: クエリパラメータ付き
```
?method=template&step=brand
?method=template&step=store
?method=template&step=emptype
?method=template&step=emp
?method=external&step=store
?method=external&step=emptype
?method=external&step=emp
```

| パラメータ | 値 | 効果 |
|---|---|---|
| `method` | `template` | テンプレート方式の画面で起動 |
| `method` | `external` | 外部連携方式の画面で起動 |
| `step` | `brand` | 業態・ブランドのアップロード画面 |
| `step` | `store` | 店舗のアップロード画面 |
| `step` | `emptype` | 雇用区分のアップロード画面 |
| `step` | `emp` | 従業員のアップロード画面 |

URL例:
```
/import?method=template&step=brand  → テンプレート > 業態・ブランドアップロード画面に直接遷移
/import?method=external&step=store  → 外部連携 > 店舗マッピング画面に直接遷移
```

##### F-2. 運用設定画面（既登録更新）からの起点（方式選択を表示）

各設定画面（brand-settings 等）の「📥 CSVインポート ↗」からの起点は **方式選択を必ず表示**。

URL: クエリパラメータなし
```
/import
```

**理由**: 既に登録済みデータの更新・追加では、ユーザーが状況に応じてテンプレート / 外部連携を都度選びたい。例えば:
- 既存ブランドの店舗を1件だけ追加 → テンプレート方式が便利
- 別の勤怠サービスから一括移行 → 外部連携方式が便利

##### F-3. 2つの起点の差別化（実装まとめ）

| 起点 | URL | インポート画面の表示 |
|---|---|---|
| **新規登録ガイド (b0)** | `?method=X&step=Y` 付き | 方式選択スキップ → 該当ステップ画面 |
| **運用設定画面** | クエリパラメータなし | 方式選択画面（テンプレ/外部）を表示 |

**理由**: ガイド側で既に「テンプレート / 外部連携」を選択済みなのに、インポート画面でも再選択を要求するのはUX重複。クエリパラメータでスキップさせて、ガイド順にスムーズにインポートできるようにする。一方、運用時は柔軟性を残すためデフォルトの方式選択UIを維持する。

**現状のモック**: 既存インポートモック (rakmy-timerecord-import-spec) はクエリパラメータ未対応のため、現状は b0 起点でも方式選択画面が表示される。本番実装時にクエリパラメータ対応すること。

#### G. 管理者データ インポート対応

管理者は新規登録ガイドの4エンティティ（業態・店舗・雇用区分・従業員）に含まれないが、運用更新時の一括登録ニーズがあるため、別途インポート機能を提供。

##### G-1. 対応範囲

| 取り込み方式 | 管理者インポート | 理由 |
|---|---|---|
| テンプレートで取り込み | ✓ 対応 | ラクミー専用テンプレートで管理者を一括登録 |
| 外部勤怠サービスから取り込み | ❌ 非対応 | 外部勤怠サービス（KoT/ジョブカン等）に「管理者」概念が存在しないため |

##### G-2. テンプレート列定義

ファイル名: `ラクミー_インポートテンプレート_⑤管理者.tsv`

| 列名 | 必須 | 入力形式 | 制約 |
|---|---|---|---|
| 管理者名 | ★ | テキスト | 1行につき1名 |
| メールアドレス | ★ | テキスト | 有効なメールアドレス形式、招待メール送信先 |
| 担当店舗 | ★ | テキスト | ② 店舗の店舗名と一致、複数指定は半角カンマ区切り |
| 権限 | ★ | 選択 | オーナー / 管理者（通常は「管理者」） |

##### G-3. 2つの起点（既存パターンと整合）

| 起点 | UI | リンク先 |
|---|---|---|
| **管理者一覧 (D-1)** ヘッダー | 「📥 CSVインポート ↗」ボタン | インポート画面 `#tpl`（テンプレ画面） |
| **インポート画面 `s-tpl1`** | テンプレ選択肢に「⑤ 管理者」 | （同画面内で選択） |

##### G-4. 依存関係

- 管理者は **店舗（② 店舗テンプレ）が登録済み** であることが前提
- 担当店舗列は既存の店舗名を参照（マッピングで紐付け）
- インポート時は店舗存在チェックを実行

##### G-5. 新規登録ガイド（b0）における取り扱い

管理者は **新規登録ガイドのインポートステップには含めない**:
- 理由: 新規登録時はオーナー1人で開始、管理者は運用開始後に追加するのが自然
- 運用開始後、管理者管理画面（D-1）から CSV インポート可能

#### H. インポート画面の「新規登録モード」と「運用更新モード」の切替（重要）

##### H-1. 設計方針: 同一画面 + URLハッシュ切替

インポート画面は **新規登録モード** と **運用更新モード** で同じ画面を使用しつつ、URLハッシュで切替える。完全別スクリーン化は採用しない（保守コスト過大）。

##### H-2. URLハッシュ仕様

| ハッシュ | モード | バナー表示 | 用途 |
|---|---|---|---|
| `#saas-setup` | 新規登録 | **あり**「📋 新規登録ガイド ・ ↩ 登録ガイドへ戻る」 | 新規登録ガイド b0 から遷移 |
| `#tpl-setup` | 新規登録 | **あり**「📋 新規登録ガイド ・ ↩ 登録ガイドへ戻る」 | 新規登録ガイド b0 から遷移 |
| `#saas` | 運用更新 | **なし** | 運用設定画面 / 直接アクセス |
| `#tpl` | 運用更新 | **なし** | 運用設定画面 / 直接アクセス |
| `#` (なし) | 運用更新 | **なし** | 通常起動（方式選択トップ画面） |

##### H-3. バナー仕様（新規登録モード時のみ表示）

- 配置: `.main` 内のメインコンテンツ先頭、`position: sticky; top: 0`
- 全インポート画面（s-saas1 → s-saas-map-edit → s-saas2 → s-saas-preview → s-saas4 等）でスクロール時も常時表示
- 内容:
  - 左: 📋 アイコン
  - 中央: 「新規登録ガイド ・ Step 2: 外部勤怠サービスからインポート」等の状況表示
  - 右: 「↩ 登録ガイドへ戻る」緑塗ボタン
- ボタン挙動: `window.close()` 試行 → フォールバックで b0 URL に遷移

##### H-4. 起点別リンクの設計

| 起点 | リンク先 | バナー表示 |
|---|---|---|
| b0 「外部連携インポート画面を開く」 | `#saas-setup` | あり |
| b0 「管理者をテンプレで取込（後追加項目）」 | `#tpl-setup` | あり |
| D-1 管理者一覧「📥 CSVインポート」 | `#tpl` | **なし**（運用更新の起点） |
| その他運用設定画面の「📥 CSVインポート」 | `#tpl` または `#saas` | **なし**（運用更新の起点） |

##### H-5. 本番実装で必要な対応

1. **クエリパラメータ + ハッシュ対応**: `applyHashRouting()` のロジックを本番にも実装
2. **バナー非表示時の幅・余白調整**: 運用更新モードではバナー分の余白を取らない
3. **完了画面（s-saas4 等）からの自動リダイレクト（任意）**: 新規登録モードで完了したらb0へ自動戻り（または案内）
4. **アクセシビリティ**: バナーに `role="banner"` / `aria-label="新規登録ガイド経由のインポート"` 推奨
5. **ブラウザバック対応**: `hashchange` イベントで適切に状態同期

##### H-6. なぜ「完全別スクリーン」にしないか

| 案 | 評価 | 採用 |
|---|---|---|
| 同一画面 + URLハッシュ切替（採用） | 保守容易、業界標準、要件達成 | ✅ |
| 完全別スクリーン（s-saas1-setup 等を別div） | コード重複、保守2倍 | ❌ |
| 別ファイル化（import-setup.html） | 保守3倍、URLレベル分離 | ❌ |

業務SaaS（freee、SmartHR、KoT等）も同様に「文脈バナー方式」を採用しており、業界標準パターン。

### 9.5 モックでの擬似連携体験

静的モックの制約を踏まえ、**デモ用シミュレーションボタン** を b0 のインポートパネルに設置:
- 「① 業態・ブランド完了」「② 店舗完了」「③ 雇用区分完了」「④ 全項目完了」の4ボタン
- クリックでインポート完了をシミュレート、進捗が該当ステップまで更新される
- エンジニアは「インポートモックを別タブで動かす → 戻り後ボタン押下 → ガイド進捗反映を確認」できる
