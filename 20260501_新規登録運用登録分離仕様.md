## 1. 設計コンセプト

タイムレコード利用開始に必要な4項目（業態・ブランド／店舗／雇用区分／従業員）の登録について、**「新規登録モード」と「運用登録モード」を画面・UIフィードバック・順序強制度で完全分離** します。

### 設計原則

| 観点 | 新規登録モード | 運用登録モード |
|---|---|---|
| **目的** | 利用開始のための4項目必須登録 | 既存運用での追加・編集 |
| **順序** | 必須（① → ② → ③ → ④） | 任意 |
| **強制度** | 強誘導（CTAモーダル） | 軽量示唆（CTAトースト） |
| **画面** | `setup-*` 専用ページ（ウィザード式） | `*-settings` 通常設定画面（一覧+追加） |
| **UI** | クリーンなステップカード | 一覧表 + 「+ 追加」 |

## 3. 画面仕様

### 3.1 B-0 登録ガイド (`#b0`)

**役割**: 4項目登録の進捗可視化・順序提示・ステップ画面への誘導ハブ

**表示内容**:
- 進捗サマリ（円形リング・パーセント・残数）
- ステップマーカー（①②③④）
- 動的ステータスメッセージ（進捗別5パターン）
- 4ステップカード（前提登録/件数/状態バッジ/CTA）

**CTAボタン**:
- 「登録する →」 → 該当 setup-* 画面へ遷移＋モーダル自動展開
- 「再編集 →」（完了済みステップ）→ 同じく setup-* へ
- 順序違反は disabled でガード

**完了後（done=4）**:
- 上部に「🎉 初期登録は完了しています」サマリバナー
- 「打刻記録を開く →」CTA
- サイドバーから「📋 登録ガイド」項目は非表示

### 3.2 setup-* 新規登録ステップ画面（4種）

各画面共通レイアウト:
- **サイドバー**: 「📋 登録ガイド」+ 「初期登録ステップ」(①②③④進捗マーク付き)
- **メインエリア**:
  - 「← 登録ガイドへ戻る」リンク
  - 中央に **緑枠ステップカード**: STEP X/4 バッジ + タイトル + 説明 + 前提登録/件数情報 + 「+ 登録する」大型ボタン

**動作**:
- 「+ 登録する」 → 該当登録モーダル展開
- モーダル登録完了 → CTAモーダル（橋渡し）→ 次の setup-* へ

### 3.3 *-settings 運用登録画面（4種）

各画面共通レイアウト:
- **サイドバー**: タイムレコ通常メニュー（打刻記録/勤務時間集計/給与/設定セクション）
- **メインエリア**:
  - タイトル + 「+ 追加」ボタン
  - 一覧表（既登録レコードを表示）

**動作**:
- 「+ 追加」 → 該当登録モーダル展開
- モーダル登録完了 → トーストCTA（右上）→ 任意で次の登録に連鎖

### 3.4 b-step-modal-* 登録モーダル（4種）

各エンティティ専用の純粋な登録フォーム。新規/運用 共通で利用。

| モーダル | フィールド |
|---|---|
| ① 業態・ブランド | 業態select / ブランド名 / ブランドコード |
| ② 店舗 | 店舗名 / 所属ブランド select / 住所 / 営業時間 |
| ③ 雇用区分 | 区分名 / 種別 / 時給単位 |
| ④ 従業員 | 名前 / 電話 / メール / 雇用区分select / 所属店舗select |

**仕様**:
- 純粋な登録フォームのみ（前提充足チェック警告は表示しない）
- 「キャンセル」「✕」 → モーダルクローズ
- 「登録」 → 進捗保存 + モーダルクローズ + モード別CTA表示

### 3.5 CTAモーダル（橋渡しモーダル）

**新規登録モード時のみ表示**。中央オーバーレイで強誘導。

```
       ✓
✓ ◯◯を登録しました
新しい◯◯を登録しました。
続けて、◯◯も登録しますか？

[次の登録に進む →]   [登録ガイドへ戻る]
```

| 登録項目 | メッセージ | 主CTA | 副CTA |
|---|---|---|---|
| ① ブランド | 続けて、このブランドの店舗も登録しますか？ | 店舗の登録に進む → | 登録ガイドへ戻る |
| ② 店舗 | 続けて、この店舗の従業員も登録しますか？ | 従業員の登録に進む → | 登録ガイドへ戻る |
| ③ 雇用区分 | 続けて、この区分の従業員も登録しますか？ | 従業員の登録に進む → | 登録ガイドへ戻る |
| ④ 従業員 | 招待メールは従業員一覧の招待送信ボタンから | 従業員一覧へ → | 登録ガイドへ戻る |

### 3.6 CTAトースト

**運用登録モード時のみ表示**。画面右上スライドイン。

```
✓ ◯◯を登録しました
新しい◯◯を登録しました。続けて関連項目も追加できます。

[続けて◯◯を登録 →]   [閉じる]
```

**動作**:
- 8秒で自動消去
- マウスホバー → タイマー停止
- ホバー解除 → 3秒タイマーで再開

## 4. ナビゲーションフロー

### 4.1 新規登録フロー（順序必須）

```
ログイン直後
   ↓
b0 登録ガイド
   ↓ 「① 登録する →」
setup-brand に遷移 + ブランドモーダル自動展開
   ↓ 「登録」
CTAモーダル「店舗の登録に進む →」「登録ガイドへ戻る」
   ├─ 「進む」 → setup-store + 店舗モーダル自動展開
   └─ 「戻る」 → b0 (進捗確認)
   ↓ 連鎖継続
setup-store → setup-emptype → setup-emp
   ↓ 全完了
完了サマリ → 打刻記録ダッシュボード
```

### 4.2 運用登録フロー（順序任意）

```
通常運用中
   ↓ サイドバー > 設定 > 業態・ブランド
brand-settings に遷移
   ↓ 「+ 追加」
ブランドモーダル展開
   ↓ 「登録」
画面右上にトースト「続けて店舗を登録 →」「閉じる」
   ├─ 「続けて」 → store-settings + 店舗モーダル自動展開（連鎖）
   ├─ 「閉じる」 / ✕ / 自動消去 → そのまま brand-settings に留まる
```

## 5. 状態管理

### 5.1 進捗状態（sessionStorage）

```js
b0Done   = 0..4   // 完了ステップ数
b0Counts = [n,n,n,n]  // 各ステップの登録件数
```

### 5.2 モード判定

```js
const isInitialMode = b0LoadDone() < 4;
const isOperationalMode = b0LoadDone() >= 4;
```

### 5.3 順序ガード

- `setup-N` が `done + 1` 以外なら警告バナー表示 + ボタン disabled
- `setup-N` で N <= done なら「再編集」モードで動作
- `b0CompleteStep(N)` 内で N !== done+1 の登録は alert で阻止

## 7. 実装にあたり確認事項（エンジニア向け）

### 7.1 ルーティング

- `/setup/brand`, `/setup/store`, `/setup/emptype`, `/setup/emp` （新規登録モード）
- `/settings/brand`, `/settings/stores`, `/settings/emptypes`, `/settings/employees` （運用モード）
- 初回ログイン時は `/setup/brand` から自動開始
- 全項目登録完了後は `/setup/*` URLは無効化（→ `/dashboard` リダイレクト）

### 7.2 モード判定ロジック

- バックエンド側で「初期登録完了フラグ」を保持するか、各テーブルの件数で判定するか
- 推奨: `companies.onboarding_completed_at` カラムを追加し、4項目すべてに1件以上の時刻を保存

### 7.3 順序ガード

- フロント側ガード: 順序違反URLにアクセス → 正しいステップへリダイレクト
- バックエンド側ガード: 必要な前提が未登録なら API側で 422 エラー返却

### 7.4 CTAモーダルの中断UX

- 「登録ガイドへ戻る」を選んだ場合、`b0` 画面で進捗バー＋次ステップ強調で再開できる
- 中断状態を sessionStorage（or DBに）に保存

### 7.5 トーストの実装

- アニメーション: スライドイン（ease 350ms）
- ホバー停止: `mouseenter` でタイマークリア、`mouseleave` で 3000ms タイマー再設定
- アクセシビリティ: `role="status"` `aria-live="polite"` 推奨

### 7.6 招待メール送信（従業員登録）

- 登録時に自動送信ではなく、**従業員一覧の「招待送信」ボタンから手動送信** が確定仕様
- 招待メールテンプレート: 既存 C-4 を「招待送信ボタン経由」に修正済み
