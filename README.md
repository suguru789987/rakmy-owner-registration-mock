## 構成する3つの画面系

このリポジトリには役割別の画面モック3系統が収録されています。

### 1. 🔑 オーナー/本部管理者 画面系
- オーナー直接登録フロー(従来: 開発者がCLI手動実行 → 自動化)
- 店舗追加/編集/削除
- 管理者・従業員の直接登録(初期PW設定→口頭共有、または招待メール)
- 給与・経営管理・外部連携設定

### 2. 👥 管理者(店舗管理者) 画面系
- 担当店舗のみの従業員登録・管理
- 勤怠承認
- 担当店舗従業員のPW再発行
- 店舗情報の編集(担当店舗のみ)

### 3. 👤 従業員(打刻者) 画面系 ★見逃されがち
- **招待メール受信 → PW設定 → ログイン → 打刻開始** の完全フロー
- 出勤/退勤ボタンの打刻画面(モバイル前提UX)
- 招待リンク有効期限 **72時間** の仕様
- 「招待リンクが無効です」エラーハンドリング
- 打刻忘れ時の管理者連絡フロー
- パスワード忘れ/再設定フロー
- 自分の勤務記録確認画面
- 従業員登録モーダル(v1→v5で**登録モーダルと編集モーダルの統一デザイン化**を実施)
- 交通費入力時の**赤枠ハイライトUX**(店舗変更時の入力忘れ防止)

## ハイライト

- オーナー → 管理者 → **従業員** の**3階層権限マトリクス**を設計
- プロダクト内用語を「**店長**」→「**管理者**」に統一する命名ルール断行
- 招待メール**72時間有効期限**、PW8文字以上の仕様
- **UIモック5バージョン**(v1.0 → v5) を並存管理
- **CS申し送り表** 7ファイルを役割別/ケース別に整備
- **ヘルプページ全面改訂**: 役割別(オーナー/管理者/**従業員**) × ケース別 のIA再設計
- 従業員アプリ向けヘルプに「はじめての方へ」「招待メールが届かない」「打刻を忘れた」等のトラブルシューティング
- **FAQ整備**: CS問い合わせ傾向から項目化
- **費用実績入力モーダル**(地代家賃実績 等) の独立モックを追加
